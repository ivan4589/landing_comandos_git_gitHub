Guía rápida de Git y GitHub

Flujo básico para subir tu proyecto a GitHub

Una guía visual, limpia y moderna para inicializar tu proyecto, hacer commits y publicarlo en GitHub.

01

Inicializar proyecto

Crea la carpeta del proyecto, entra en ella e inicializa Git.

Terminal 
mkdir proyecto-Git
cd proyecto-Git
git init
02

Primer commit local

Agrega los archivos al área de preparación y registra tu primer cambio.

Terminal 
git add .
git commit -m "Descripción de lo que hicimos"
03

Crear repositorio en GitHub

Ingresa a GitHub desde tu navegador y crea un repositorio remoto para vincular tu proyecto local.

04

Vincular repositorio remoto

Conecta tu proyecto local con el repositorio que creaste en GitHub.

Terminal 
git remote add origin URL_DEL_REPOSITORIO_REMOTO
05

Enviar el proyecto a GitHub

Haz el primer push para subir tu proyecto al repositorio remoto.

Terminal 
git push -u origin master
06

Actualizar con nuevos cambios

Cada vez que modifiques tu proyecto, repite este flujo.

Terminal 
git add .
git commit -m "Nuevos cambios"
git push origin master
Trabajo con ramas

Comandos para crear y manejar ramas en Git

01

Verificar qué ramas existen

Comprueba si ya existe alguna rama en tu repositorio.

Terminal 
git branch
02

Crear una nueva rama

Crea una nueva rama para hacer cambios sin afectar la rama principal.

Terminal 
git branch nombre-de-la-rama
03

Cambiar a la nueva rama

Después de crearla, cámbiate a esa rama para trabajar allí.

Terminal 
git switch nombre-de-la-rama

Al cambiarte a la nueva rama, tendrás una copia del código actual de la rama principal para poder modificarlo.

04

Integrar cambios a la rama principal

Cuando tu trabajo esté listo, vuelve a la rama principal y luego integra la rama que creaste.

Terminal 
git switch main
git merge nombre-de-la-rama

Nota: para hacer el merge, primero debes estar ubicado en la rama principal.
